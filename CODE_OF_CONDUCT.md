# Code of Conduct

## Our Pledge

We want Anvil to be a practical, respectful open-source project. Contributors
and maintainers are expected to communicate with professionalism, assume good
intent where reasonable, and keep discussions focused on improving the project.

## Expected Behavior

- Be respectful when giving or receiving feedback.
- Keep issue and pull request discussions technical and actionable.
- Credit related work and avoid misrepresenting ownership.
- Report security issues through the security policy instead of public issues.

## Unacceptable Behavior

- Harassment, intimidation, or personal attacks.
- Publishing private information or credentials.
- Spam, repeated off-topic promotion, or bad-faith disruption.
- Using project spaces to coordinate abuse or illegal activity.

## Enforcement

Maintainers may edit, hide, or remove disruptive content and may restrict
access for contributors who repeatedly violate this code. Security reports,
legal concerns, and moderation issues should be sent through the private
channels described in [SECURITY.md](./SECURITY.md).
